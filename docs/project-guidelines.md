# Project Guidelines

- Mobile-/Browser-freundliche Webbasis.
- Erst lokal im Browser testbar machen.
- Keine privaten Daten oder Zugangsdaten ins Repo.
- Aenderungen klein und nachvollziehbar halten.
- Vor Aenderungen immer `git status`.
- Nach Codeaenderungen Tests ausfuehren.
- Erst nach erfolgreichen Tests committen.
- Assets spaeter bevorzugt ueber Aseprite CLI/Exportpipeline erzeugen.
