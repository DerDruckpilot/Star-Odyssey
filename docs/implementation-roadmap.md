# Implementation Roadmap

Diese Reihenfolge ist die empfohlene Planung fuer die naechsten PRs. Bugfixes und kleinere Korrekturen koennen jederzeit dazwischen liegen.

1. PR 1: Dokumentation/Projektdefinition, dieser PR.
2. PR 2: Hauptmenue mit Sprache Toggle und UI-Grundstruktur.
3. PR 3: Neues-Spiel-Flow mit Spieleranzahl und QR-Code-Platzhaltern.
4. PR 4: Spielfeld-Layout als statisches Board mit Sternenhintergrund und Hex-/Raumpunktstruktur.
5. PR 5: Erste eigene PNG-Assets fuer Planetentypen und UI.
6. PR 6: Datenmodell fuer Spieler, Ressourcen, Planeten, Raumpunkte und Verbindungen.
7. PR 7: Runden-/Zug-State-Machine.
8. PR 8: Wuerfel-/Ertragsphase.
9. PR 9: Handels-/Bauphase als erste UI.
10. PR 10: Schiffbewegung ueber Raumpunkte.
11. PR 11: Erkundung verdeckter Systeme.
12. PR 12: Kolonie-/Stationsgruendung.
13. PR 13: Save/Load.
14. PR 14: Controller-/QR-/Second-Screen-Prototyp.
15. PR 15: TV-/Fire-TV-Wrapper erst spaeter.

## Leitlinie

Jeder PR soll einen klaren, testbaren Schritt liefern. Erst Struktur und Datenmodell, dann Regeln, dann Interaktion, dann Politur.
